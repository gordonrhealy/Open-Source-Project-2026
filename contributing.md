Contributing to Financial Tracker
Thank you for your interest in contributing to the Financial Tracker project. This document explains how to report issues, request features, submit pull requests, and set up the project locally.
Getting Started
Before contributing, please:
• 	Read the README for an overview of the project.
• 	Check existing issues to avoid duplicates.
• 	Ensure your contribution aligns with the project’s goals: simplicity, clarity, and reliability.
How to Set Up the Project Locally
1. Clone the repository: https://github.com/gordonrhealy/Open-Source-Project-2026
2. Install dependencies: npm install
3. Start the development server: npm run dev
   
How to Report a Bug
When opening a bug report, include:
• 	A clear title
• 	Steps to reproduce
• 	Expected vs actual behaviour
• 	Browser/device information
• 	Screenshots (if relevant)
Use the Bug Report issue template when available.
How to Request a Feature
Feature requests should explain:
• 	What problem the feature solves
• 	How it improves the user experience
• 	Any alternatives you considered
Use the Feature Request issue template.

Making Changes (Pull Requests)
1. 	Create a new branch
2. 	2. 	Make your changes following the project’s coding style.
3. 	Run tests (if available) and ensure everything passes.
4. 	Commit using clear, descriptive messages
5. 	Push your branch
6  	Open a Pull Request and include:
• 	What the PR does
• 	Linked issue (e.g., “Fixes #12”)
• 	Screenshots for UI changes
• 	Testing instructions
Code Style Guidelines
• 	Use consistent formatting (Prettier/ESLint if configured)
• 	Keep functions small and focused
• 	Write meaningful variable and function names
• 	Avoid committing commented‑out code
Commit Message Format
Use conventional commits:
• 	 new feature
• 	 bug fix
• 	 documentation changes
• 	 formatting only
• 	 code restructuring
• 	 adding or updating tests
• 	 maintenance tasks

Code of Conduct
Be respectful, constructive, and collaborative. Harassment or discrimination of any kind is not tolerated.
Thank You
Your contributions help improve the project for everyone. Whether you’re fixing a bug, improving documentation, or adding a new feature, your effort is appreciated.
